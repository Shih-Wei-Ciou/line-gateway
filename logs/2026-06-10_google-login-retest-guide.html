    Google 登入 重新測試 步驟卡

    你之前已經在 Web App 點 Google 登入成功過。一次性設定（Google Console + .env）都做完了，這份只走「重新驗證」的最短路徑 —— 確認它現在還活著。

    設定是一次性的；成功過就代表設好了。這次只是重新跑一遍，看端到端有沒有斷掉。

    前提（之前已完成，這次只是確認還在）

    你記得登入成功過，代表下面這些應該都還在。這次不用重做，只要快速掃一眼沒被改掉就好：

    要確認的東西預期值
    Google Console OAuth Client還在，redirect URI 仍是 http://localhost:5001/api/members/oauth/google/callback
    Test users 名單你要測的那個 Gmail 還在裡面
    backend/.env仍有 GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET / GOOGLE_CALLBACK_URL
    WEBAPP_URL 的 port= 前端實際開的 port（native dev 通常是 http://localhost:3000）
    不用再碰 Google Console。 Console 設定是一次性的，成功過就代表設好了。除非你刪過 OAuth Client 或換了 callback 網址，否則這次直接從下面「重新測試」開始。

    重新測試 · 五步

    Step 01 · 啟動後端

    要能讀到 .env 的 Google 變數。Docker 跑法：

    docker-compose up -d yuanyin-backend
# 若這次改過 .env 或 google_login.py，要加 --build：
# docker-compose up -d --build yuanyin-backend

    確認後端活著（沒帶 token 回 401 是正常的，代表服務有起來）：

    curl http://localhost:5001/api/members/me
    Step 02 · 啟動前端
    cd C:\templeuser\frontend
npm run dev

    看它印出的網址（通常 http://localhost:3000）。這個 port 必須等於 .env 裡的 WEBAPP_URL，否則會「登入成功但跳空白頁」。

    Step 03 · 開無痕視窗進「我的」頁

    按 Ctrl+Shift+N 開無痕 → 進 http://localhost:3000 → 點到「我的」頁。

    一定要用無痕。 一般視窗的瀏覽器擴充（趨勢科技 URL 掃描）會吃掉 OAuth 的 state cookie，害你卡在 invalid_state —— 這是你 LINE 登入時踩過的同一個雷。

    Step 04 · 按「使用 Google 登入」

    會被導去 Google 授權畫面 → 選你的 Gmail（Test users 裡那個）→ 按「繼續 / 允許」。

    Step 05 · 確認登入狀態

    導回前端時網址會閃過 ?login_success=true&token=eyJ...，接著「我的」頁應顯示你的 Google 名字 + 頭像。重整頁面（F5）名字還在 = token 有存進 localStorage，端到端通過。

    成功長這樣

    網址出現 login_success=true → 「我的」頁顯示 Google 名字 → 重整後仍在。三個都對 = Google 登入端到端正常，可以接著開 PR 合回 main。

    如果這次出錯

    大多是環境問題，不是程式問題。對照症狀直接套解法：

    症狀原因解法
    ?login_error=invalid_state 瀏覽器擴充吃掉 state cookie 改用無痕視窗
    ?login_error=server_error
    後端 log 有 Failed to resolve oauth2.googleapis.com    		 Docker 容器 DNS 連不到 Google 重啟 Docker Desktop（WSL2 DNS 復原）
    登入成功但跳空白頁 / 連不到 WEBAPP_URL port ≠ 前端 port 改 .env 的 WEBAPP_URL 成前端實際 port，重啟後端
    Google 報 redirect_uri_mismatch Console redirect URI 跟 GOOGLE_CALLBACK_URL 不一致 兩邊都要 http://localhost:5001/api/members/oauth/google/callback
    名字沒出來，但網址有 token 前端沒接到 URL token 確認 MemberProvider 有讀 URL token（已實作）；清快取重試
    重整後名字消失 token 沒存進 localStorage DevTools → Application → Local Storage 看有沒有 lyz_member_token
    表 1 · 六個常見症狀。前兩個（invalid_state / server_error）是你 LINE 登入時就踩過的，Google 同款。
