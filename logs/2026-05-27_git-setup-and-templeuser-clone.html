關鍵實作決策

Decision · 為什麼用「選項 A：把 .git 搬上層」
比較 A/B/C 三個方案後選 A（搬 .git 而非重 init 或搬程式碼）。 理由：最少動到檔案、保留 GitHub remote、保留 Initial commit 歷史、 不需要 force push（避免養成壞習慣）。
Caveat · 差點再次外洩 token 的危險
發現 .env.broken-backup 含真實 LINE token（172 字元）。 第二次差點推上 GitHub。教訓：任何 backup 檔案都要先檢查內容、 再決定刪不刪、絕不能無腦 git add .。
Decision · 八個分批 commit、不是一次 commit
依「邏輯主題」分：bootstrap / models / app+webhook / services / templates+static / scripts / docs+logs / md。 未來查歷史時可以精準回到某個功能加入的時間點。
Decision · clone 學長 repo 用兄弟資料夾結構
C:\templeuser\ 跟工作目錄是平行的，不放進 line-gateway repo。 理由：這是別人的 repo、有獨立 git 歷史、塞進去會把自己 repo 變肥幾十倍。 未來協作就用「兩個本地兄弟資料夾」+「兩個 GitHub repo」的模式。
Open · Web App 後端怎麼分工還未定
今天還沒決定。本來想用「我做後端 / 學長做前端」分， 但發現 Next.js 14 App Router + RSC 架構下，前後端綁在同一個檔案、 無法用「前後端」切分。改用「子系統切」（身分系統 / 訂閱系統 / 廟宇瀏覽 / 報名）， 但這需要學長同意，週三會議才會確認。
