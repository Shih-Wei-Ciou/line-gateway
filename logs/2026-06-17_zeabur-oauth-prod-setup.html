    關鍵缺口 — 登入按鈕還是壞的

    已解決 · 正式站登入成功 · 2026-06-17

    真正的根因是 前端 Dockerfile 沒宣告 NEXT_PUBLIC_BACKEND_URL 這個 build arg → 就算 Zeabur 設了變數，build 也收不到 → 按鈕 fallback 到 localhost:5001。修法：Dockerfile 加 ARG/ENV + docker-compose build args（commit 94ca137）→ push main → Zeabur 重 build → 前端服務的 NEXT_PUBLIC_BACKEND_URL=https://yuanyinb.zeabur.app 才烤得進去。重 build 後正式站 LINE / Google 登入皆實測成功。

    通用教訓：Next.js 的 NEXT_PUBLIC_* 是 build 時烤死的。要在容器化部署生效，必須三者齊備：① Dockerfile 宣告對應 ARG；② 該服務環境變數有設；③ 觸發重 build（不是重啟）。少一個都不會生效。
    前端服務新增這一條，問題就解：
    NEXT_PUBLIC_BACKEND_URL = https://yuanyinb.zeabur.app（單 b）
    ⚠️ NEXT_PUBLIC_ 是 build 時烤進去 → 設完前端要手動 Redeploy 一次。
