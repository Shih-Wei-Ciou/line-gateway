關鍵實作決策

Decision · 從 6 格改 4 格
舊 6 格（我的平安符、定位參拜、線上祈福、神明小故事、福報商城、帳號設定）反映 「LINE 為主」的舊架構。新架構 LINE 變招牌、Web App 才是主場，所以 menu 重新定義為 4 個入口（關於我們 / 拜拜通 / 最新消息 / 合作管道），全部跳 Web App。
Decision · line-bot-sdk v3 雙 client
v3 的 Rich Menu API 分成 MessagingApi（建立 / 設 default / 刪除） 和 MessagingApiBlob（上傳圖片），兩個都要用同一個 Configuration 初始化。
Note · --confirm 安全機制
不加 --confirm 只做 dry run（生成圖片），不會打 LINE API。 Cleanup 會列出所有將被刪除的 rich menu id，逐一印出 "Deleting: richmenu-xxx"。
Caveat · URL 改了要重建 menu
Rich Menu 一旦建立，URL 寫死在 LINE 平台上，改 .env 沒用。 週三跟 Web App 對齊網址後必須重跑一次 setup_rich_menu.py --confirm。
Note · 字型 fallback
Windows 用微軟正黑體 Bold (msjhbd.ttc)，Linux 用 NotoSansCJK， macOS 用 PingFang。都找不到時 fallback 到 PIL default font（會醜但會跑）。

  • 週三定案 WEBAPP_BASE_URL 後重跑腳本
