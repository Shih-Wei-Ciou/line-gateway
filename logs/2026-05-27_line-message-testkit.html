任務目標

測試 LINE Messaging API 能做的所有訊息類型，先把六種樣式都做出來、實際推到手機看， 再決定哪些保留、哪些拿掉。這是一份「實驗包」而不是正式功能 — Flex Carousel、Quick Reply、Loading Animation、Location、Sticker 全部一次驗過。

架構上把訊息組裝（flex_templates.py）跟推播流程（messages.py） 分開，再透過 POST /admin/test/send 一個 endpoint 觸發六種測試 type， 不用每次重啟 server 改 code。

關鍵實作決策

Decision · 全部用 push 不用 reply
Webhook 既有的 echo handler 用 reply_message（reply token 1 次性、免費）。 新版改全部走 push_message：(1) admin endpoint 沒有 reply token、必須 push； (2) welcome sequence 要在 loading animation 之後再送訊息，超過 reply token 時效； (3) messages.py 集中一套 push 邏輯，admin 跟 webhook 共用。 測試階段 push 訊息量小，billing 不是問題。
Caveat · Loading Animation 用 raw HTTP
line-bot-sdk 3.5.0 還沒把 show_loading_animation 包進 SDK （已驗 dir(messaging) + dir(MessagingApi) 都沒有）。 改用 requests.post 直接打 https://api.line.me/v2/bot/chat/loading/start。 失敗時 log warning 但不擋整個 welcome 流程 — loading 是錦上添花，掛了也能往下走。
Design · 莊重襯線、白底黑字
卡片用 #FFFFFF 底、#1A1612 標題、#6A6058 副標、 #1A1612 黑底白字按鈕、4px 圓角、無陰影。 刻意不跟 LINE 預設綠色 / 圓潤 UI 同調 — 這是「廟」的 tone，不是「電商」的 tone。 卡片 1 加「→ →」滑動引導，提示這是 Carousel。
Design · 關鍵字 routing 用 substring + 優先序
「拜拜通」跟「拜拜」會撞在一起：前者要進 Carousel，後者要進拜拜指南。 解法：用 in 比對（substring）+ 一個 priority 排序的 routes 表， 「拜拜通」放最高優先 → 先被吃掉。 效果：使用者打「我想看拜拜通」進 Carousel，「教我拜拜」進指南卡。 無關鍵字命中就沉默，避免每句話都被機器人回（會超煩）。
Decision · 三官寶殿座標用近似值
Location Message 是要測「LINE 內建地圖卡」的 UI，不是要做正式廟宇定位。 用台南白河區一組合理近似座標 (23.3528, 120.4150) + 看起來像真地址的字串。 等之後接 TEP DB 拿到實際座標，再做真實版的 Location push。
Caveat · Loading Animation 只對「主動聊過」的對象有效
LINE 官方限制：使用者必須先在聊天室發過至少一則訊息給官方帳號， loading animation 才會出現。剛 follow 的新用戶在 follow event 觸發時可能還沒聊過 → loading 那一輪 API 可能回 400，但不影響後續 4 則訊息推送。

怎麼測（給未來的你）

# 1. 填 .env 兩個變數後重啟 server
python app.py

# 2. ngrok 對到 :5002（另開一個視窗）
ngrok http 5002

# 3. curl 六種測試 type（PowerShell 版）
$key = (Get-Content .env | Select-String '^ADMIN_KEY=').ToString().Substring(10)
foreach ($type in 'explore','welcome','quick_reply','location','sticker','loading_test') {
  Write-Output "--- $type ---"
  Invoke-RestMethod -Method Post -Uri http://localhost:5002/admin/test/send `
    -Headers @{ 'X-Admin-Key' = $key } `
    -ContentType 'application/json' `
    -Body (@{ type = $type } | ConvertTo-Json)
  Start-Sleep -Seconds 2
}
