## 八、重要決策記錄(別忘了)

| 決策 | 為什麼 |
|------|--------|
| dev mode 用 X-Line-Uid header | Postman 好測,正式版要改 access token |
| 訂閱用軟刪除(is_active=false) | 保留歷史,重訂閱可以「復活」 |
| TEP 不需要 user_id 綁定 | `/api/public/*` 直接吃 lineUserId(今天 explore 才發現) |
| TEP_API_KEY 暫不帶 | TEP 目前沒驗證,future-proof 先留環境變數 |
| Internal API 用 X-Gateway-Key 驗證 | 防止任何人觸發推播 |
| flex_builder 視覺先寫一版 | 結構先定,5/8 後重做視覺、結構不動 |
| postback data 用 query string 格式 | LINE 標準,易解析 |
