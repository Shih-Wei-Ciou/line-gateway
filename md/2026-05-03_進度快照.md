## 七、重要決策記錄(別忘了)

| 決策 | 為什麼 |
|------|--------|
| dev mode 用 X-Line-Uid header | Postman 好測,正式版要改成 access token 驗證 |
| 訂閱用軟刪除(is_active=false) | 保留歷史,重訂閱可以「復活」 |
| TEP 不需要 user_id 綁定 | `/api/public/*` 直接吃 lineUserId(今天 explore 才發現) |
| TEP_API_KEY 暫不帶 | TEP 目前沒驗證,future-proof 先留環境變數 |
| 不做 Flex 卡片 builder | 等設計稿來了再做(避免重做) |
