一套你可以一直沿用的報告設計

這份檔案同時是「成品範例」和「零件箱」。下次要寫任何實驗紀錄、評估報告、專案總結，直接複製它、改字、換配色就好，不用每次重新想版面。

把力氣花在內容上，版面交給範本。一致的格式本身就是一種專業感。

怎麼用這份範本

就四步，照做不會錯:

Step 01 · 複製

整份 HTML 複製一份，改 <title> 和封面那段文字，就是你新報告的起點。

Step 02 · 換配色(可選)

只改最上面 :root 裡的變數。例如把 --accent 換成別的色，整頁的章節編號、連結、重點數字會一起跟著變，不用一個一個改。

Step 03 · 填內容

每加一個章節，就複製一個 <section> 區塊，給它 id 和 data-title。需要表格、KPI、圖表時，到下面元件庫挑一個複製過去改數字。

Step 04 · 不用管目錄

左側目錄是程式自動生的 —— 它會掃所有 section 當主章節、掃裡面的 <h3> 當子項，還自動做「點擊跳轉、捲動高亮、子項折疊」。你只要專心寫內容。

給 Claude Code 用: 也可以把這份檔案丟給 Claude Code 當設計基底，跟它說「照這份範本的設計風格與元件，幫我做 XX 報告」，它就會沿用同一套色彩、字體、版面。

色彩系統

三層表面

核心觀念: 頁面不是只有一個底色。由暗到亮分三層,層與層一定要有看得出來的明度差,東西才不會糊成一片。改報告時最常出包的就是這裡 —— 別讓兩層太接近。

L0 頁面底--l0
L1 章節容器--l1
L2 元件卡片--l2

強調色與語意色

整頁只用一個主強調色(陶土橙),負責章節編號、連結、最佳值、重點數字。其餘狀態交給三個語意色:綠=好、黃=注意、紅=壞。別把語意色拿來當裝飾,不然又花了。

強調 Accent--accent
好 Good--good
注意 Warn--warn
壞 Bad--bad

字體系統

三種角色分工:襯線撐標題與引文的個性、無襯線負責好讀的內文、等寬專門對齊數字(tabular-nums,讓上下位數對齊)。

結論 / 重點提示

用顏色快速分辨語氣。語氣中性用預設,正面加 .good,要注意加 .warn,壞消息加 .bad。寫法盡量口語,像在跟人解釋。

結論 · 好消息

分割模型量出來的覆蓋率,比框選法準很多 —— 誤差只有 0.35~1.71 個百分點。

結論 · 要注意

有一個模型在連續影格裡會「忽抓忽不抓」,跳變率 33%,不太適合直接拿去跑即時影片。

結論 · 壞消息

100 張本來不該框的背景照,模型有 80 張還是手癢框了 —— 這就是為什麼要把它們加進訓練當反例。
